# Usability Test Protocol for DocSign

**Target User Group:** Adults aged 65 and older
**Application:** DocSign - Electronic Document Signing
**Version:** 1.0
**Last Updated:** December 2024

---

## 1. Participant Criteria

### Primary Requirements
- Age 65 years or older
- Able to read English
- No known cognitive impairments that would prevent task completion
- Willing to provide verbal feedback during testing

### Technology Comfort Levels (Recruit Mix)
Recruit participants across these three categories for balanced insights:

| Level | Description | Target Count |
|-------|-------------|--------------|
| **Beginner** | Uses smartphone/computer mainly for calls, email, or basic browsing. Rarely or never signs documents electronically. | 3-4 participants |
| **Intermediate** | Comfortable with common apps (email, banking, shopping). May have signed documents electronically 1-2 times. | 3-4 participants |
| **Experienced** | Regularly uses multiple apps and services. Has signed documents electronically multiple times. | 2-3 participants |

### Screening Questions
1. What is your age?
2. How often do you use a computer or smartphone? (Daily / Weekly / Rarely)
3. Have you ever signed a document electronically (like DocuSign, Adobe Sign)? (Yes / No)
4. If yes, how would you rate that experience? (Easy / Somewhat difficult / Very difficult)
5. Do you wear glasses or have any vision corrections?
6. Do you have any conditions affecting your hands (arthritis, tremor)?

## 2. Testing Environment Setup

### Equipment Required
- [ ] Computer or tablet with DocSign loaded (test environment)
- [ ] External mouse (in addition to trackpad for preference)
- [ ] Large external monitor (if available, minimum 15" screen)
- [ ] Notepad and pen for facilitator notes
- [ ] Timer or stopwatch
- [ ] Printed feedback forms
- [ ] Recording device (with participant consent)
- [ ] Backup device in case of technical issues

### Room Setup
- Well-lit room with no glare on screen
- Quiet environment with minimal distractions
- Comfortable seating with proper screen height
- Water available for participant
- Tissues available

### Technical Setup Checklist
- [ ] Browser set to 100% zoom (or higher if participant prefers)
- [ ] Test PDF document pre-loaded in session
- [ ] Network connection stable
- [ ] Test signing link ready: `sign.html?session=test&recipient=r1&key=testkey`
- [ ] Clear browser cache and cookies before each session
- [ ] Audio/screen recording software ready (if consented)

### Test Document
Use a simple 1-2 page PDF with:
- Clear signature field location
- Initials field
- Date field
- Readable text (14pt+ font)

## 3. Task List for Participants

Participants will complete these tasks in order. Read each task aloud and allow the participant to work independently.

### Task 1: Open the Signing Page
**Instructions:** "You received an email asking you to sign a document. Click the link in the email to open the signing page."

**Success Criteria:**
- Page loads without error
- Loading indicator is visible and understood

**Time Limit:** 2 minutes

---

### Task 2: Review Document Information
**Instructions:** "Before signing, look at the information shown. Tell me who sent this document and what the document is called."

**Success Criteria:**
- Correctly identifies sender name
- Correctly identifies document name
- Understands the consent text

**Time Limit:** 3 minutes

---

### Task 3: View the Document
**Instructions:** "Click the button to review the document. Scroll through to see all pages if there are multiple pages."

**Success Criteria:**
- Successfully clicks "Review Document" button
- Document viewer opens
- Can scroll through document pages
- Can identify signature field locations

**Time Limit:** 3 minutes

---

### Task 4: Draw a Signature
**Instructions:** "Click on the signature box and draw your signature using your mouse, trackpad, or finger."

**Success Criteria:**
- Opens signature modal
- Successfully draws a signature
- Applies the signature to the field

**Time Limit:** 5 minutes

---

### Task 5: Use Typed Signature Option
**Instructions:** "Now let's try a different way. Clear your signature and try typing your name instead. Choose a font style you like."

**Success Criteria:**
- Switches to "Type" tab
- Types name in text field
- Previews signature with different fonts
- Applies typed signature

**Time Limit:** 4 minutes

---

### Task 6: Complete the Signing Process
**Instructions:** "Now finish signing the document by clicking the finish button."

**Success Criteria:**
- Clicks "Finish Signing" button
- Understands completion confirmation
- No confusion about next steps

**Time Limit:** 2 minutes

---

### Task 7 (Optional): Test Offline Mode
**Instructions:** "We're going to simulate losing internet connection. Continue using the app and tell me what happens."

**Success Criteria:**
- Notices offline indicator
- Understands that work is saved locally
- No panic or confusion

**Time Limit:** 3 minutes

## 4. Metrics to Collect

### Quantitative Metrics

| Metric | Description | How to Record |
|--------|-------------|---------------|
| **Task Completion Rate** | Did participant complete the task successfully? | Yes / No / Partial |
| **Time per Task** | Seconds from task start to completion | Stopwatch |
| **Error Count** | Number of mistakes, wrong clicks, or confusion moments | Tally marks |
| **Assistance Requests** | Number of times participant asked for help | Tally marks |
| **Satisfaction Rating** | Participant's rating after each task | 1-5 scale |

### Qualitative Metrics
- Verbal feedback during tasks (think-aloud)
- Facial expressions and body language
- Points of hesitation or confusion
- Spontaneous comments
- Post-task reflections

## 5. Rating Scales

### Task Difficulty Rating (Ask after each task)
"On a scale of 1 to 5, how difficult was this task?"

| Rating | Label | Description |
|--------|-------|-------------|
| 1 | Very Easy | Completed without any difficulty |
| 2 | Easy | Minor hesitation but completed smoothly |
| 3 | Moderate | Some confusion but figured it out |
| 4 | Difficult | Struggled significantly, needed hints |
| 5 | Very Difficult | Could not complete without help |

### Overall Satisfaction Rating (Ask at end)
"On a scale of 1 to 5, how satisfied are you with this signing experience?"

| Rating | Label |

| 1 | Very Dissatisfied |
| 2 | Dissatisfied |
| 3 | Neutral |
| 4 | Satisfied |
| 5 | Very Satisfied |

## 7. Important Notes for Facilitators

### Do's
- Speak clearly and slowly
- Allow extra time for responses
- Remain patient and encouraging
- Take detailed notes on observations
- Ask clarifying questions when needed

### Don'ts
- Do NOT help unless participant is truly stuck (give hints first)
- Do NOT show frustration or impatience
- Do NOT rush the participant
- Do NOT lead the participant with suggestions
- Do NOT make assumptions about their abilities

### When to Intervene
- After 2 failed attempts at the same action
- If participant expresses significant distress
- If participant explicitly asks for help
- After time limit expires without progress

### Intervention Protocol
1. First: Ask "What are you trying to do?"
2. Second: Give a general hint ("Look at the top of the screen")
3. Third: Give a specific hint ("Click the blue button that says Review")
4. Fourth: Demonstrate the action and note as "failed with assistance"

## 9. Success Criteria for the Application

The application passes usability testing if:

1. **Task Completion Rate:** 80% or higher across all tasks for all participants
2. **Critical Task Success:** 100% completion rate for Tasks 1, 4, and 6 (core signing flow)
3. **Average Time:** No task exceeds 2x the expected time limit on average
4. **Satisfaction:** Average overall satisfaction rating of 3.5 or higher
5. **Error Recovery:** All participants can recover from at least one error without assistance

If these criteria are not met, document specific failure points and prioritize fixes before release.
